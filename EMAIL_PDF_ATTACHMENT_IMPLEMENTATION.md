# Email PDF Attachment Implementation

## Overview
When creating or deleting an invoice or order and confirming that an email should be sent to the customer, the same PDF that is generated for PDF download is now attached to the email.

## Implementation Details

### Backend Changes

#### 1. smtp-phpmailer.php
- Added `$attachment` parameter to `sendEmailPHPMailer()` function
- Supports attachments in format: `['data' => base64_string, 'filename' => string]`
- Uses PHPMailer's `addStringAttachment()` to attach PDF files
- Binary data is base64-encoded for email MIME transmission

#### 2. send-approved-emails-inline.php
- Extracts `attachment` field from email data
- Passes attachment to `sendEmailPHPMailer()` function

### Frontend Changes

#### 3. email-notifications.js
- **New function**: `generatePDFAttachment(documentType, documentData)`
  - Generates PDF using the same `generatePDF()` function used for downloads
  - Uses standard template (5th parameter = true) for customer-facing documents
  - Converts PDF to base64 using jsPDF's `output('datauristring')` method
  - Generates filename in format: `Rechnung_ID_YYYY-MM-DD.pdf` or `Auftrag_ID_YYYY-MM-DD.pdf`
  - Returns: `{data: base64String, filename: string}` or `null` on error

- **Updated function**: `queueAndSendImmediately(type, data, attachment = null)`
  - Added optional `attachment` parameter
  - Includes attachment in email payload when present

- **Updated notification functions** (all accept optional `fullDocument` parameter):
  - `notifyNewInvoice()` - Generates and attaches invoice PDF
  - `notifyNewOrder()` - Generates and attaches order PDF
  - `notifyInvoiceDeleted()` - Generates and attaches deleted invoice PDF
  - `notifyOrderDeleted()` - Generates and attaches deleted order PDF

#### 4. UI Files (auftraege-ui.js, rechnungen-ui.js, auftraege-render.js, rechnungen-render.js)
- Updated calls to notification functions to pass full document object as second parameter
- This ensures the PDF has all necessary data for generation

## How It Works

1. **User creates/deletes invoice or order**
2. **User confirms email notification** via browser dialog
3. **PDF is generated** using the same function as the download button
4. **PDF is converted** to base64 string
5. **Email is prepared** with PDF attachment data
6. **Email is sent** via PHP backend using PHPMailer
7. **PDF is attached** to the email using MIME base64 encoding

## Testing Instructions

### Prerequisites
- Email configuration must be set up in `backend/config.json`
- SMTP credentials must be valid
- Email notifications must be enabled in settings

### Test Cases

#### Test 1: New Invoice Email with PDF
1. Navigate to Rechnungen (Invoices) page
2. Click "Neue Rechnung" button
3. Fill in invoice details and items
4. Save the invoice
5. Confirm "Yes" when asked to send email notification
6. **Expected**: Email is sent with invoice PDF attached

#### Test 2: New Order Email with PDF
1. Navigate to Aufträge (Orders) page
2. Click "Neuer Auftrag" button
3. Fill in order details and items
4. Save the order
5. Confirm "Yes" when asked to send email notification
6. **Expected**: Email is sent with order PDF attached

#### Test 3: Deleted Invoice Email with PDF
1. Navigate to Rechnungen page
2. Click delete button (minus icon) on an existing invoice
3. Confirm deletion
4. Confirm "Yes" when asked to send email notification
5. **Expected**: Email is sent with deleted invoice PDF attached

#### Test 4: Deleted Order Email with PDF
1. Navigate to Aufträge page
2. Click delete button (minus icon) on an existing order
3. Confirm deletion
4. Confirm "Yes" when asked to send email notification
5. **Expected**: Email is sent with deleted order PDF attached

#### Test 5: Decline Email
1. Perform any of the above actions
2. Click "Cancel" when asked to send email notification
3. **Expected**: No email is sent, action completes normally

### Verification
- Check recipient's email inbox for received email
- Verify PDF is attached to the email
- Open PDF attachment and verify it contains correct document data
- Verify PDF matches the one that would be generated by the download button
- Check `backend/smtp-debug.log` for email sending logs

## Error Handling
- If PDF generation fails, email is sent without attachment
- If email sending fails, notification remains in queue
- Errors are logged to browser console and `backend/smtp-debug.log`

## Security Considerations
- PDF data is base64-encoded for transport
- PHPMailer handles MIME encoding securely
- No security vulnerabilities detected by CodeQL scanner
- Standard template is used for customer-facing documents

## Known Limitations
- PDF attachment size is limited by email server settings (typically 10-25 MB)
- Large PDFs may take longer to generate and send
- Email sending is synchronous and may delay the UI response

## Future Improvements
- Add progress indicator for PDF generation
- Implement asynchronous email sending
- Add retry logic for failed email sends
- Allow user to preview PDF before sending email
